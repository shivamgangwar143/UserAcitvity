User Activity Data Pipeline

A production-grade Medallion Architecture ETL pipeline processing millions of events using PySpark and PostgreSQL.

Apache Spark Python PostgreSQL Docker Power BI

Medallion Architecture Flow

🥉

Bronze Layer

Raw Data Ingestion

Ingests synthetic CSV data (simulating 2M+ rows) into a Data Lake. Stored as raw Parquet files with ingestion timestamps to maintain an immutable history.

🥈

Silver Layer

Data Cleaning & Transformation

Handles data quality issues: removes duplicates, normalizes mixed timestamp formats, filters negative watch times, and fills null values. Partitioned by date for performance.

🥇

Gold Layer

Business Aggregations

Computes highly optimized metrics: Daily Active Users (DAU), watch-time per content, region engagement, and device distribution datasets.

📊

Serving Layer

PostgreSQL & Power BI

Loads Gold layer Parquet data into a PostgreSQL Data Warehouse via JDBC, making it instantly queryable for Power BI dashboards and Ad-hoc SQL.

Key Engineering Features

🛠️ Data Quality & Transformations

  • Dynamic parsing of 4 different timestamp formats.
  • Deduplication across 2+ million rows.
  • Handling missing categorical data (Device, Region).
  • Removing logically invalid data (negative watch time).

⚡ Performance Optimizations

  • Columnar Storage: Parquet + Snappy compression reduces size by ~70%.
  • Partition Pruning: Silver layer partitioned by Date to prevent full-table scans.
  • Spark Tuning: Optimized shuffle partitions and coalesced outputs for single-node efficiency.
  • Incremental Loading: Uses high-water-mark timestamps to process only new data.
